Design notes — Material 3 sur écran auto 15.6"

Adaptations clés pour le format paysage automotive

Touch target min.
56 × 56 dp
vs 48 dp standard M3 — réduit les erreurs en conduite
Type minimum
14 sp body / 16 sp button
Lecture rapide à 60-80 cm
Layout primary
Nav Rail (96 dp)
Toujours visible — pas de bottom nav (mains sur volant)
Master / Detail
2-column split
Exploite la largeur 1920px — pas besoin de naviguer profond
Status persistant
Top bar
Heure + clim + batterie + signal toujours visibles
Theme
Dark only
Conduite de nuit = priorité absolue. Day mode synchronisé véhicule

Composants Material 3 utilisés

  • Navigation Rail — 96 dp à gauche, 5 destinations, toujours visible (équivalent voiture du bottom nav mobile)
  • Top App Bar Small avec actions contextuelles à droite + horloge
  • FAB Extended — action principale (ex. "Lancer mirror")
  • Filled / Tonal / Outlined / Text Buttons — 56 dp de hauteur (vs 40 standard)
  • Icon Buttons 56×56 avec état toggled (background secondaryContainer)
  • Cards filled / outlined avec tonal elevation (pas d'ombre)
  • List items avec leading icon circulaire 48 dp et trailing widget
  • Material Switch M3 (track + thumb anim, plus large que mobile)
  • Filter / Assist Chips — 44 dp pour faciliter le tap en mouvement
  • Search bar docked (corner pill, surfaceContainerHigh)
  • Slider avec thumb 24 dp et badge de valeur
  • Tabs primary avec indicator coloré

Patterns spécifiques DashCast

  • Main 2-column : grille d'apps (gauche, 60%) + statut cluster + aperçu temps réel (droite, 40%). Tu n'as plus besoin de switcher d'écran pour voir l'état du mirror.
  • Cluster preview live : reproduction mini du rendu cluster (1920×720) directement dans le panneau de droite, mise à jour en temps réel.
  • Diag master/detail : liste des tests à gauche, résultat structuré à droite (mono-box syntax highlighted). Plus de scroll pour voir le résultat.
  • Settings 2-column : type/overscan à gauche, comportement/about à droite — tout visible sans scroll sur les options principales.
  • SysInfo dashboard tiles : 4 cartes overview en haut, listes affichages/services en dessous, rapport monospace à droite. Lecture en un coup d'œil.
  • Log table-like : colonnes alignées (level / timestamp / tag / message) avec border-left coloré selon niveau. Parsing visuel rapide.
  • Status persistant top-right : clim + batterie + signal + heure, sans interférer avec l'écran clim natif BYD.

Stratégie de migration (rappel)

  1. Bump androidx.appcompat:1.1.0 → 1.6.1 + ajout com.google.android.material:material:1.11.0
  2. Création themes.xml avec parent Theme.Material3.DayNight.NoActionBar (variant dark forcée par défaut)
  3. Import de la palette ci-dessus dans colors.xml + values-night/colors.xml (mais theme par défaut = dark vu le contexte)
  4. Refonte des layouts existants pour exploiter la largeur (split panes via ConstraintLayout avec guidelines à 60%)
  5. Ajout du NavigationRailView en remplacement des boutons toolbar actuels
  6. Remplacement Button → MaterialButton, Switch → MaterialSwitch, CardView → MaterialCardView
  7. Hauteur de boutons : créer un style Widget.DashCast.Button.Automotive avec minHeight=56dp
  8. Substitution AlertDialog.Builder → MaterialAlertDialogBuilder
  9. Test visuel sur device + ajustements (probablement padding et density-specific)

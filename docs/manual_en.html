1. Overview

DashCast is an Android application that lets you project any app from the infotainment screen onto the digital instrument cluster of your BYD vehicle. Navigation, music, videos — anything running on the centre screen can be redirected to the driver-facing cluster display.

  • ✅ Compatible with BYD Seal EU (DiLink 3.0, firmware Di3.0 / 6125F)
  • ✅ No system modification required
  • ✅ Local ADB over TCP (localhost) — no PC needed once set up
  • ✅ Automatic detection when an app is closed externally
💡 Prerequisite: Enable TCP ADB debugging in Settings → Developer options → Wireless debugging (or "ADB over network"). This is a one-time step. On first DashCast launch, an "Allow USB debugging?" popup appears — tap Always allow from this computer.

5. During Projection — Control Panel

When an app is active on the cluster, a dark panel appears at the bottom of the main screen with three remote-control features:

5.1 Mirror (📺 Mirror)

Tap 📺 Mirror in the status bar to display a live copy of the cluster content inside DashCast. You can interact with this copy by touch — events are forwarded to the cluster display.

The mirror uses SurfaceControl to capture the display. If the mirror is unavailable, an automatic screenshot every 2 seconds is used as fallback.

5.2 Split Mode (⬛⬛ Split)

Tap ⬛⬛ Split to share the cluster between two apps:

  • Full screen — One app occupies the entire cluster
  • ⬜⬛ Left (50%) — Main app on the left, second app on the right
  • ⬛⬜ Right (50%) — Main app on the right

5.3 Hide Panel

Tap Hide ▼ to collapse the control panel and see the full app list.

7.1 Cluster Type

Select the screen size of your instrument cluster. For the BYD Seal EU, select 12.3 inches (cmd=30).

7.2 Display Margins (Overscan)

Adjust margins to frame content within the visible area of the cluster screen. Curved-glass clusters often have physical edges that extend beyond the usable display area.

  • Left / Right — Horizontal margin (0–200 px on each side)
  • Top / Bottom — Vertical margin (0–200 px top and bottom)

Click Apply now to see the result immediately if an app is currently projected. Values are saved between sessions.

💡 Recommended defaults for the Seal EU: Left/Right = 80 px, Top/Bottom = 50 px.

9. FAQ & Troubleshooting

❓ The "Allow USB debugging?" popup does not appear

Check that TCP ADB debugging is enabled in the infotainment developer settings. If the option is missing, first enable developer mode (tap the build number 7 times in About).

❓ The app does not appear on the cluster after selection

  • Make sure you tapped Activate Projection before selecting the app.
  • Some apps refuse to launch on a secondary display. Check the Logs for the error message.
  • Try closing and reopening DashCast, then repeat the sequence.

❓ Content is clipped or offset on the cluster

Adjust the display margins in ⋮ → Settings. Increase Left/Right if content overflows horizontally, Top/Bottom if it overflows vertically. Click Apply now to see the result immediately.

❓ The "← Main" and "✕" buttons remain visible after closing the app

DashCast automatically detects app termination (via /proc monitoring). If the UI is stuck, tap Stop Projection to force a reset.

❓ After restarting the car, do I need to reconfigure everything?

No. The cluster type and overscan margins are saved. The last projected app is also remembered. Only the ADB connection may require tapping Activate Projection again.
